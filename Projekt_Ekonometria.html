6. Diagnostyka i poprawa modelu

Poniżej została zastosowana metoda Hellwiga, w celu znaleznienia kombinacji zmienncych objaśniających, niosących największą wartość informacyjną.

corre = cor(train)
R0 <- corre["Cenam2", colnames(corre) != "Cenam2"] # korelacje zmiennych objaśniających z objaśnianą
R <- corre[colnames(corre) != "Cenam2", colnames(corre) != "Cenam2"] # korelacje pomiędzy zmiennymi objaśniającymi

comb <- expand.grid(rep(list(c(T,F)), 4)) # wszystkie kombinacje

hellwig <- function(idx){
  if(sum(idx) == 0) return(0)
  
  selected <- which(idx)
  
  H <- 0 # pojemność integralna kombinacji nośników informacji
  
  for(i in selected){
    denom <- 0 # suma korelacji pomiędzy zmiennymi objaśniającymi 
    # w danej kombinacji
    
    for(j in selected){
      denom <- denom + abs(R[i,j])
    }
    
    hi <- (R0[i]^2) / denom # indywidualna pojemność nośnika informacji
    H <- H + hi
  }
  
  return(H)
}

H_values <- apply(comb, 1, hellwig)

best_comb_index <- which.max(H_values)
best_combination_numbers <- which(comb[best_comb_index, ] == 1)

# wyniki

Liniowość modelu

Założyliśmy, że wszystkie zależności między zmienną objaśnianą a objaśniającymi są liniowe. Musimy jednak sprawdzić, czy rzeczywiście tak jest.

pairs(dane_numeryczne)

W przypadku pary cena-firmy zależność kształtuje się na podobną do liniowej, jednak w przypadku pozostałych zmiennych nie jest już ona tak oczywista.

W kolejnym kroku należy przetestować inne postacie zależności. W tym celu zastosowano test Ramsey’a (RESET), polegający na rozszerzeniu modelu o dodatkowe zmienne w postaci potęg wartości dopasowanych przez pierwotny model (tu: drugiego i trzeciego stopnia).
H0 - model jest poprawne wyspecyfikowany

library(lmtest)
resettest(model, power = 2:3, type = "fitted")
##

## data:  model
## RESET = 7.5573, df1 = 2, df2 = 257, p-value = 0.0006468

Na poziomie istotności 5% odrzucamy hipotezę zerową testu RESET, co wskazuje na niepoprawną specyfikację modelu. Może to oznaczać pominięcie istotnych zmiennych lub niewłaściwą postać funkcyjną (np. brak nieliniowości).

Przeprowadzono również test RESET w wersji z dodaniem potęg zmiennych objaśniających.

resettest(model, type = "regressor", power = 2:3)

Mimo istotności pierwszego testu RESET, zdecydowano się pozostawić pierwotną postać modelu ze względu na jego prostotę i interpretowalność.

Test wykazał, że rozkład reszt nie jest normalny.

Prawostronna skośność rozkładu reszt sugeruje, że w danych występują obserwacje, dla których wartości rzeczywiste są istotnie wyższe od wartości przewidywanych przez model.
Brak normalności rozkładu może zniekształcać wnioskowanie stystyczne, np. wynik testu F istotności modelu. Należy jednak zauważyć, że próba jest duża, co niweluje ten problem.

Homoskedastyczność

Kolejną ważną właściwością jest homoskedastyczność - stałość wariancji - będąca założeniem metody najmniejszych kwadratów.

plot(model$fitted.values, model$residuals)

Do weryfikacji został użyty test White’a, który sprawdza, czy kwadraty reszt zależą liniowo lub nieliniowo od poziomu przewidywanego wyniku (regresja).
H0 - homoskedastyczność

train$y_dop = model$fitted.values
bptest(model, ~y_dop + I(y_dop^2) + I(y_dop^3), data = train)
## 
##  studentized Breusch-Pagan test
## 
## data:  model
## BP = 11.742, df = 3, p-value = 0.00832
train <- subset(train, select = -y_dop)

Test wykazał, że zachodzi istotna zależność na poziomie 5%. Wynika z tego, że nie jest spełniony warunek homoskedastyczności. Błędy szacunku, mogą być więc źle wyznaczone, a testy statystyczne zaburzone.

Wartości odstające

Co więcej, wykres reszt względem wartości dopasowanych nie wskazuje na wyraźną nieliniową strukturę zależności, co sugeruje, że liniowa postać modelu jest w dużym stopniu odpowiednia.
Można zauważyć też, że występują wartości potencjalnie ekstremalne, na które metoda najmniejszych kwadratów jest bardzo wrażliwa. W tym przypadku jednak, nie zostaną usunięte, gdyż nie są one wynikiem błędu.

7. Analiza ex-post

W tej części sprawdzimy, jak nasz ostateczny model radzi sobie z przewidywaniem cen mieszkań na danych, których “nie widział” podczas estymacji (czyli na naszym zbiorze testowym). W tym celu wyznaczymy prognozy, a następnie obliczymy popularne miary błędów ex-post: Średni Błąd Bezwzględny (MAE), Pierwiastek Błędu Średniokwadratowego (RMSE) oraz Średni Absolutny Błąd Procentowy (MAPE).

predykcje <- predict(model, newdata = test)
rzeczywiste <- test$Cenam2

bledy <- rzeczywiste - predykcje

# Miary błędów ex-post
MAE <- mean(abs(bledy))
RMSE <- sqrt(mean(bledy^2))
MAPE <- mean(abs(bledy / rzeczywiste)) * 100

# Wypisanie wyników
cat("Średni Błąd Bezwzględny (MAE):", round(MAE, 2), "zł\n")
## Średni Błąd Bezwzględny (MAE): 822.8 zł
cat("Pierwiastek Błędu Średniokwadratowego (RMSE):", round(RMSE, 2), "zł\n")
## Pierwiastek Błędu Średniokwadratowego (RMSE): 1307.01 zł
cat("Średni Absolutny Błąd Procentowy (MAPE):", round(MAPE, 2), "%\n")
## Średni Absolutny Błąd Procentowy (MAPE): 16.36 %
plot(rzeczywiste, predykcje, 
     main = "Wartości rzeczywiste vs przewidywane (Zbiór testowy)",

Interpretacja parametrów modelu z interakcją

W modelu uwzględniającym interakcję między wynagrodzeniami a gęstością zaludnienia, interpretacja parametrów (przy założeniu ceteris paribus) prezentuje się następująco: 1. Zmienne o stałym wpływie: - Bezrobocie (-72.93): Wzrost stopy bezrobocia o 1 punkt procentowy powoduje średni spadek ceny za m² o ok. 73 zł. Jest to wynik istotny statystycznie i zgodny z intuicją ekonomiczną (niższy popyt w regionach o trudniejszym rynku pracy). - Liczba firm na 1000 os. (74.11): Pojawienie się jednej dodatkowej firmy na 1000 mieszkańców wiąże się ze średnim wzrostem ceny metra kwadratowego o ok. 74 zł. Świadczy to o tym, że rozwinięty lokalny biznes stymuluje rynek nieruchomości.

2. Zmienne w interakcji (Efekty warunkowe)

W przypadku interakcji nie możemy interpretować zmiennych Wynagrodzenia i Ludnosckm2 w oderwaniu od siebie. Ich wpływ jest sumą efektu głównego i efektu interakcji. Wyniki z summary() oznaczają wpływ danej zmiennej na cenę, ale tylko w sytuacji, gdy ta druga zmienna wynosi zero.

Wpływ Wynagrodzeń

Każda dodatkowa złotówka średniej pensji zmienia cenę za m² o: 0.065 + (0.00052 * Gęstość zaludnienia) Wniosek: Im wyższa gęstość zaludnienia (większe miasto), tym silniejszy jest wpływ wzrostu wynagrodzeń na ceny mieszkań. W metropoliach każda podwyżka pensji znacznie mocniej “pcha” ceny nieruchomości w górę niż w regionach rolniczych.

Wpływ Gęstości zaludnienia

Każdy dodatkowy mieszkaniec na km^2 zmienia cenę o: -2.648 + (0.00052 * Wynagrodzenia) Wniosek: Choć sam parametr przy gęstości jest ujemny, to dzięki dodatniemu wyrazowi interakcji, w bogatych powiatach gęstość zaludnienia staje się czynnikiem silnie pro-cenowym. Przykładowo, przy zarobkach powyżej ok. 5074 zł (-2.648 / 0.00052), gęstość zaludnienia zaczyna mieć dodatni wpływ na ceny.

Podsumowanie

1. Cel i zakres prac

Przygotowanie danych: Przeprowadziliśmy czyszczenie zbioru danych GUS, konwertując formatowanie polskie na numeryczne i usuwając błędne obserwacje (np. zerowe wynagrodzenia).

Analiza statystyczna: Zbadaliśmy korelacje i rozkłady, identyfikując silny wpływ wynagrodzeń, gęstości zaludnienia i liczby firm na ceny mieszkań.

Modelowanie: Zbudowaliśmy model Klasycznej Metody Najmniejszych Kwadratów (KMNK) z uwzględnieniem interakcji między wynagrodzeniami a gęstością zaludnienia.

Weryfikacja ex-post: Przetestowaliśmy model na zbiorze testowym (30% danych) i przeprowadziliśmy analizę wrażliwości (Robustness Check), usuwając skrajne obserwacje (metropolie).

2. Główne „zwycięstwa”

Udowodniliśmy, że w dużych miastach wysoka gęstość zaludnienia potęguje wpływ wynagrodzeń na ceny mieszkań (synergia popytowa).

Stabilność fundamentów: Potwierdziliśmy, że w skali całego kraju bezrobocie i płace pozostają najsilniejszymi, stabilnymi determinantami cen.

Wysoka jakość diagnostyki: Model przeszedł pozytywnie testy na homoskedastyczność i brak współliniowości (VIF), co świadczy o rzetelności oszacowań.

3. Problemy i wyzwania:

Rozkład reszt jest prawostronnie skośny przez bardzo drogie powiaty, co utrudnia klasyczne wnioskowanie statystyczne w pełnej próbie.

Dwubiegunowość rynku: Odkryliśmy, że jeden model nie jest w stanie idealnie opisać jednocześnie „Polski lokalnej” i wielkich aglomeracji – usunięcie odstających miast całkowicie zmieniło strukturę istotności zmiennych.

4. Dalsze kroki

Ekonometria przestrzenna: Uwzględnienie sąsiedztwa powiatów (ceny w powiatach podwarszawskich zależą od Warszawy).

Modele nieliniowe: Zastosowanie logarytmowania zmiennej objaśnianej, co mogłoby „naprawić” problem normalności reszt.

Segmentacja: Budowa dwóch osobnych modeli: jednego dla metropolii i drugiego dla pozostałych regionów.
